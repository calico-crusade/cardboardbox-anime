Hi, I'm Cardboard

I am a software developer by trade, and I work in health insurance as my daily job, but I do various volunteer and side projects to sate my boredom. Most of the projects I do are open-source and can be found on my github. They are mostly related to my reading addiction, Artificial Intelligence, or helpful libraries for other developers. My primary programming languages are C# and TypeScript, but I've dabbled in many others.

I read A LOT, in 2022 I read over 400 books, and I've already read over 100 books this year (as of July 2023). I mostly read light novels, but I also read some non-fiction books and text-books. I'm a big fan of the "In another world" (Isekai) genre, but I also enjoy romance, rom-coms, and slice-of-life.

While most people would probably label me a weeb, that isn't entirely accurate. I'm not really interested in Japan as a whole, I just like the stories and shows that come from there. But enough about me, you probably want to know more about this site.

What is this site and why?

This website (CardboardBox - Anime), and subsequent applications, are mostly a pet project of mine and is something I made to sate my boredom. I have no intention of making money off of any of this, and I don't plan on adding ads ever (seems like too much work). I mentioned before that I read alot, and most of this website (now-adays) is to facilitate that. However it didn't begin that way.

Originally, the purpose of this website was to keep track of my anime collection (what I've watched, and where I watched it). I used a few different streaming sites at the time (funimation, crunchyroll, VRV, hidive, netflix, hulu, etc), and there was no easy way to search all of them at once. Hence this website was born.

It was setup to scrape the various streaming site APIs and store all of the data into a postgres database. From there I could create lists of what I've watched and where (Here's some rom-coms for you!). A great side-effect was that I could also search all of the sites, keep track of where the dubbed and subbed versions were, and recommend shows to friends.

Nowadays

In 2021/22 Sony (Funimations parent company) bought Crunchyroll , and started merging libraries. They also decided to kill off VRV and Funimation, and just stick with Crunchyroll . Netflix's pricing changes were stupid. And Hulu barely updates their library with interesting shows. All of those changes kinda killed the purpose of this project as the only two sites still around are hidive and crunchyroll.

Around the same time, I stopped really watching a lot of shows, and started reading a lot more. I prefer reading novels on my iPad using the kindle app, but there are a bunch of novels that have been fan translated that aren't available via Amazon yet. So I decided to write an app to scrape the fan translations and create ebooks (epubs) for them. I then ran into an issue where I couldn't keep the epubs up-to-date as I'd have to regenerate all of them whenever they were updated. So I started loading them into a table on postgres so I could keep track of just the changes.

What about Manga?

I didn't used to read a lot of manga, I was mostly a novel kinda guy. That was until I realized that I could read ahead in some of the stories because the manga was further ahead than the novels. But all of the existing manga readers hurt my eyes! They were all so bright and white! "Why didn't you just turn on night-light or lower your brightness?" Ya don't think I tried? It was still too bloody bright! So I made my own reader that has a blue-light filter (that you can adjust) and a dark theme by default. Yay! My old eyes are saved (I'm not actually that old, I'm only in my 20s!)

I then proceeded to add a bunch of other features to it at the behest of some friends and aquaintances; reverse image searching, progress tracking, read statistics, library searching, push notifications (via a discord bot), etc. All of those ended up being a lot of work, but it was a good waste of time.

You're probably curious about the tech stack I'm using for this site, so here it is:

Front End:

There are actually 2 different front-ends for this site. The first is the site you're on. It's written in angular 14 and a load of custom CSS (I'm not a designer, so it's not pretty). You can actually see how this site has enhanced my UX/UI skills as I've worked on it, because the anime related stuff is ugly as hell. The other front-end is a Nuxt3 app that is used for the manga reader. It's a lot more standardized then this site, and is a lot prettier (also has more customization too!)

Back End:

All of the back-end APIs (yes, there are multiple) are written in asp.net core and C#. I've been using C# for almost 12 years now, and it's still my go-to language for just about anything.

Database(s):

There are actually a number of databases that I use throughout these applications. The primary database that houses all of the data for this site is Postgres. I have an apache Cassandra instance that houses my authentication application and deals with all the OAuth stuff. The bot has it's own SQLite database for handling interactions and resumability. Then there is the MongoDB database (that I'm moving away from) that used to house the light-novel stuff. And finally, there is the elastic search database that is used as a glorified vector database for the reverse image search stuff.

Hosting must cost a lot...

I host pretty much everything locally on my own servers. The image you see next to this is the rack I have in my office. It's got an ubuntu server in it where most of the web-stuff is hosted, and a windows 10 machine for the AI stuff. The other computer is my main rig, for gaming and development and whatnot.

For internet speeds, I have gigabit to my house, and 2.5gig within my local network. Upload speeds are kinda an issue, but it works for the most part. There is also a fail-over system in place that keeps the site up (most of the time) if my internet goes down. Whenever my network dies, an AWS EC2 instance gets spun up and the site continues hosting from there. It's not perfect, but it works (most of the time).

Wow, that's a major time investment

Yep. It is. But it was worth it. I've had a lot of fun and it's a great way to stave off boredom between books. The first commit to the git repo was August 3rd 2022, but I probably started working on it 3-4 months before that. I've been working on it on and off since then, and it still gets regular updates whenever I run into issues or get bored. I use this site and the manga reader every day, so I tend to notice issues pretty quickly and update it as needed otherwise it would bug me.

There are also a lot of other projects/libraries, that I've written, that this site uses. Some of the major open-source ones that are used here are:

  • CardboardBox.Database - a micro-ORM written ontop of Dapper that provides some query gen.
  • MangadexSharp - a library for interfacing with the MangaDex API (I'm a FE dev (recruit) for MD!)
  • CardboardBox.Proxy - a proxy server that allows me to bypass CORS/hot-link issues and cache images.
  • CardboardBox - a collection of extension methods and utilities that I use in most of my projects.

Some of the other libraries or APIs that I use (that I didn't write) include:

  • stable-diffusion-webui - a client used for AI Image Generation
  • nsfw-detection - an API that can be used to detect NSFW images (I cannot find the original source)
  • dsys/match - an API for handling reverse image matching
